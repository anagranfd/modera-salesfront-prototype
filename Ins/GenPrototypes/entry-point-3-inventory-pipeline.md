# Publishing Tool — Entry Point 3: Inventory Pipeline
## Screen Flow & Wireframes

---

## Контекст

Inventory Pipeline — это основной список всех машин в инвентаре.
Это единственная точка входа, где возможна **пакетная публикация**:
выбрать несколько машин сразу и опубликовать их на нескольких каналах за одно действие.

**Два независимых сценария в этой точке входа:**

- **Single vehicle** — клик по машине или кнопка Publish в строке →
  открывается модальная Publishing Panel. Логика идентична Entry Point 1.
  Описывается кратко.

- **Batch** — выбор 2+ машин через чекбоксы →
  появляется Batch Action Bar → многошаговый flow публикации пакета.
  Это главный и уникальный сценарий Entry Point 3. Описывается подробно.

# Open questions специфичные для Entry Point 3

1. **Лимит batch** — есть ли максимальное количество машин в пакете?
   При 50+ машинах матрица прогресса нечитаема — нужна ли агрегация?

2. **Пагинация при batch** — если пользователь выбрал машины
   на странице 1, перешёл на страницу 2 и выбрал ещё —
   сохраняется ли выбор между страницами?

3. **Параллельность vs последовательность** — публикация внутри пакета
   идёт параллельно по всем каналам или последовательно?
   Это влияет на дизайн прогресс-матрицы.

4. **Retry scope** — при Retry после batch: повторяются только
   упавшие операции или весь пакет заново?

5. **Pending после закрытия** — если пользователь закрыл диалог
   пока есть `◐ Pending` операции, где он увидит финальный статус?
   Нужен ли notification center или обновление точек в таблице?

6. **Конфликт состояний** — если машина уже Published на Auto24
   и попадает в batch publish на Auto24 — это republish (обновление объявления)
   или ошибка "уже опубликовано"?
