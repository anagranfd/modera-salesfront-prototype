# Publishing Tool — Entry Point 2: Vehicle Builder
## Screen Flow & Wireframes

---

## Контекст и отличие от Entry Point 1

Vehicle Builder — это большая форма редактирования данных машины.
В отличие от Vehicle Card (которая read-only с быстрым доступом),
Builder — это рабочее место для заполнения и корректировки данных.

**Ключевое отличие этой точки входа:**
Publishing Panel здесь не просто показывает статусы —
она активно связана с формой. Незаполненные поля подсвечиваются
прямо в форме, и пользователь может заполнить их не выходя из Builder.

**Что остаётся таким же, как в Entry Point 1:**
- Все 4 статуса (unpublished / pending / published / error)
- Логика выбора каналов
- Validation check перед публикацией
- Progress и result состояния
- Unpublish flow

Поэтому в этом документе повторяющиеся состояния описаны кратко со ссылкой
на Entry Point 1, а подробно расписаны только уникальные для Builder сценарии.

> **Важное архитектурное решение:**
> Publishing Panel в Builder размещается как **постоянная правая колонка**,
> а не как один из табов. Это позволяет видеть статус публикации
> и inline-подсказки одновременно с заполнением любого таба формы.
> Альтернативный вариант — таб "Publishing" — описан отдельно ниже (→ Альтернатива Б).

## А-2. Unique: Validation с inline-подсветкой полей в форме

**Это ключевой уникальный сценарий Entry Point 2.**

Пользователь нажал Publish. Validation check определил незаполненные поля.
В отличие от Entry Point 1, здесь Builder может **сразу подсветить
нужные поля прямо в форме** — не нужно переходить никуда.

```
┌──────────────────────────────────────────────────────────────────────────────┐
│  VW Golf 1.6 TDI · 2019 · WVW123456789          [ Save ]  [ Preview ]       │
├──────────────────────────────────────────────────────────────────────────────┤
│  [ Basic Info ]  [ Details ]  [ Equipment ]  [ Media ]  [● Publishing ]     │
├──────────────────────────────────┬───────────────────────────────────────────┤
│                                  │                                           │
│  BASIC INFO                      │  ⚠ Validation warnings                   │
│  ────────────────────────────    │  Fill required fields to publish          │
│                                  │  ─────────────────────────────────        │
│  Make        [Volkswagen     ]   │                                           │
│  Model       [Golf           ]   │  [✓] Auto24      ████████████ 96%        │
│  Year        [2019           ]   │                  ✓ Ready                 │
│  Body        [Hatchback   ▼  ]   │                                           │
│  Engine      [1.6 TDI 115hp  ]   │  [✓] Mobile.de   ██████████░░ 88%        │
│  Gearbox     [Manual      ▼  ]   │                  ⚠ Requires 100%         │
│  Color       [Silver      ▼  ]   │                  · Mileage               │
│                                  │                  · Fuel type              │
│  ┌───────────────────────────┐   │                  [↓ Scroll to fields]     │
│  │ ⚠  Mileage     [      ]km │   │                                           │
│  │    Required by Mobile.de  │   │  [✓] SS.lv       ████████░░░░ 74%        │
│  └───────────────────────────┘   │                  ✓ Accepts partial        │
│                                  │                                           │
│  ┌───────────────────────────┐   │  [ ] Autoplius   ██████████░░ 88%        │
│  │ ⚠  Fuel type  [Select ▼ ] │   │                  ⚠ Requires 100%         │
│  │    Required by Mobile.de  │   │                  · Interior color         │
│  └───────────────────────────┘   │                  · Doors count            │
│                                  │                  [↓ Scroll to fields]     │
│  Doors       [___]               │                                           │
│  Seats       [___]               │  ────────────────────────────             │
│                                  │  [ Back ]  [ Publish 2 of 4 ]            │
└──────────────────────────────────┴───────────────────────────────────────────┘
```

**Аннотации:**
- Поля с ошибкой получают inline-обёртку с красной рамкой и подписью
  `"Required by [Channel name]"` — прямо в форме, без перехода
- В панели справа: список каналов с полосками и незаполненными полями
- `[↓ Scroll to fields]` — если нужное поле не в текущем табе,
  скроллит форму к нужному месту или переключает таб
- `[ Publish 2 of 4 ]` — публикует только готовые каналы (Auto24 + SS.lv),
  игнорирует блокированные
- Форму можно редактировать прямо сейчас — при заполнении поля
  панель справа обновляется real-time (полоска растёт, предупреждение исчезает)

## А-5. Поле в другом табе — переключение таба

Если незаполненное поле находится не в текущем табе формы
(например, Interior color находится в табе "Details", а пользователь на "Basic Info"):

```
┌──────────────────────────────────┬───────────────────────────────────────────┐
│                                  │                                           │
│  [ Basic Info ]  [● Details ] ..  │  ⚠ Validation warnings                  │
│                                  │  ─────────────────────────────────        │
│  DETAILS                         │                                           │
│  ────────────────────────────    │  [ ] Autoplius   ██████████░░ 88%        │
│                                  │                  ⚠ Requires 100%         │
│  Engine power   [115        ] hp  │                  · Interior color         │
│  CO2 emissions  [___        ]    │                    [→ Tab: Details]       │
│  Euro standard  [Euro 6   ▼  ]   │                  · Doors count            │
│                                  │                    ← you are here         │
│  ┌───────────────────────────┐   │                                           │
│  │ ⚠  Interior color         │   │                                           │
│  │    [   Select color ▼   ] │   │                                           │
│  │    Required by Autoplius  │   │                                           │
│  └───────────────────────────┘   │                                           │
│                                  │                                           │
│  Doors  ← highlighted in form   │                                           │
│  ┌───────────────────────────┐   │                                           │
│  │ ⚠  Number of doors [___]  │   │                                           │
│  │    Required by Autoplius  │   │                                           │
│  └───────────────────────────┘   │                                           │
│                                  │                                           │
└──────────────────────────────────┴───────────────────────────────────────────┘
```

**Аннотации:**
- `[→ Tab: Details]` — кликабельная ссылка в панели, которая переключает таб формы
  и скроллит к нужному полю
- `← you are here` — подсказка для полей в текущем табе
- Таб "Details" получает визуальный индикатор (точка или бейдж)
  что в нём есть незаполненные обязательные поля

## А-6. Progress и Result

Идентичны А-4 и А-5 из Entry Point 1.
Публикация идёт прямо из Builder — пользователю не нужно
возвращаться в Vehicle Card.

```
┌──────────────────────────────────┬───────────────────────────────────────────┐
│                                  │                                           │
│  BASIC INFO                      │  Publishing...                            │
│  ────────────────────────────    │  ─────────────────────────────────        │
│                                  │                                           │
│  Make        [Volkswagen     ]   │  ✓  Auto24       ● Published              │
│  Model       [Golf           ]   │  ⟳  Mobile.de    ◐ Sending...             │
│  ...                             │  ·  SS.lv        · Waiting                │
│                                  │                                           │
│  (форма остаётся редактируемой   │  ─────────────────────────────────        │
│   во время публикации,           │  Processing 2 of 3...                     │
│   но Save заблокирован)          │  [████████████░░░░░░░░] 67%               │
│                                  │                                           │
└──────────────────────────────────┴───────────────────────────────────────────┘
```

**Аннотации:**
- Форма остаётся видимой и даже редактируемой во время публикации
- Кнопка `[ Save ]` заблокирована во время публикации (чтобы не изменить данные в процессе)
- После завершения — Save разблокируется

# ВАРИАНТ Б — Publishing как таб (альтернативный layout)

Если архитектурное решение — таб, а не колонка:

```
┌──────────────────────────────────────────────────────────────────────────────┐

├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  Publishing channels                                                         │
│  ──────────────────────────────────────────────────────────────             │
│                                                                              │
│  ○  Auto24       ○ Not published     ████████████ 96%   ✓ Ready             │
│  ○  Mobile.de    ○ Not published     ███████████░ 88%   ⚠ Missing: 2 fields │
│  ○  SS.lv        ○ Not published     ████████░░░░ 74%   ✓ Accepts partial    │
│  ○  Autoplius    ○ Not published     ██████████░░ 80%   ⚠ Missing: 3 fields │
│  ○  City24       ○ Not published     █████████░░░ 82%   ✓ Ready             │
│                                                                              │
│  ──────────────────────────────────────────────────────────────             │
│  ⚠  2 channels need more data. Fill required fields:                        │
│                                                                              │
│  Mobile.de requires:                                                         │
│    · Mileage (km)    → [ Go to Basic Info ]                                  │
│    · Fuel type       → [ Go to Basic Info ]                                  │
│                                                                              │
│  Autoplius requires:                                                         │
│    · Interior color  → [ Go to Details ]                                     │
│    · Doors count     → [ Go to Basic Info ]                                  │
│    · Reg. number     → [ Go to Details ]                                     │
│                                                                              │
│  ──────────────────────────────────────────────────────────────             │
│  [ + Publish to channels ]                                                   │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

**Аннотации:**
- Таб "Publishing" получает бейдж `⚠2` — количество каналов с проблемами
- Список незаполненных полей разбит по каналам
- `[ Go to Basic Info ]` / `[ Go to Details ]` — клик переключает таб и скроллит к полю,
  потом пользователь возвращается на Publishing таб вручную
- Недостаток по сравнению с Вариантом А: нет real-time обновления,
  нужно переключаться между табами туда-обратно

---

**Сравнение вариантов:**

| Критерий | Вариант А (колонка) | Вариант Б (таб) |
|---|---|---|
| Видимость статуса | Всегда | Только на Publishing табе |
| Real-time обновление | Да | Нет (нужно переключаться) |
| Место в UI | Забирает ~280px ширины | Не занимает постоянно |
| Сложность реализации | Выше | Ниже |
| Контекст | Лучше при частой публикации | Лучше при редкой публикации |

> Рекомендация: Вариант А предпочтительнее для пользователей,
> которые заполняют форму с целью публикации. Вариант Б — если
> публикация редкий сценарий и экономия экранного пространства важна.

# Сводная таблица состояний Entry Point 2

| # | Состояние | Что уникально vs EP1 |
|---|---|---|
| А-1 | Builder + Panel idle | Panel как колонка, всегда видна |
| А-2 | Validation + inline highlight | Поля подсвечены прямо в форме |
| А-3 | Real-time update при вводе | Панель обновляется при каждом изменении |
| А-4 | Канал разблокирован | Кнопка Publish обновляет счётчик |
| А-5 | Поле в другом табе | Переключение таба + скролл к полю |

**Состояния, идентичные Entry Point 1** (не требуют отдельного дизайна):
- Выбор каналов (EP1 → А-2)
- Publishing in progress (EP1 → А-4)
- Result: Success (EP1 → А-5)
- Mixed state / Error detail (EP1 → Б-1, Б-2)
- Unpublish flow (EP1 → В-1...В-4)

# Open questions специфичные для Entry Point 2

1. **Real-time validation** — валидация происходит на клиенте (instantly) или через API-запрос?
   Это влияет на скорость обновления полоски completeness.
2. **Save при публикации** — если пользователь заполнил поля но не сохранил,
   а нажал Publish — данные сохраняются автоматически или требуют явного Save сначала?
3. **Таб-индикаторы** — нужны ли бейджи на табах формы (Basic Info ⚠, Details ⚠)
   чтобы показать в каком табе есть незаполненные обязательные поля?
4. **Колонка vs таб** — какой вариант layout предпочтителен с точки зрения
   существующей архитектуры Builder?
5. **Ширина колонки** — достаточно ли места в текущем layout Builder для
   постоянной правой колонки Publishing Panel (~280px)?
